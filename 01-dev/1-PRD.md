# [動態生存錢包 (Dynamic Survival Wallet)] Vibe Coding 專屬產品需求文件 (PRD)

> **專案備註：** 本專案為個人自用網站，注重極簡體驗、開發效率與遊戲化制約。

## 1. 產品概述 (Project Overview)
* **核心價值：** 專注於「今日可用額度」的極簡記帳，透過自動把每日餘額轉存撲滿、將超支平攤至未來，結合生存遊戲般的分類與獎懲機制，讓存錢變成無痛的全自動闖關遊戲。
* **目標對象：** 沒有記帳習慣、容易在月底透支、想存錢但缺乏自制力的大學生與年輕小資族。（本版本為開發者自用版）
* **使用情境：** 每天早上打開確認當天預算、消費後隨手花 3 秒鐘輸入金額、晚上睡前查看存錢進度與明日額度。

---

## 2. 系統架構與技術建議 (Tech Stack for AI)
* **前端呈現：** 建議使用 React.js (透過 Vite 建立) 搭配 Tailwind CSS。使用 Framer Motion 強化動態回饋。
* **後端與資料庫：** 使用 **SQLite** 作為輕量級本地端資料庫儲存方案。透過 Vite Dev Server API 進行資料同步，確保 Local-First 的資料持久化。

---

## 3. 核心功能與邏輯 (Core Features - MVP)

### [功能區塊 A：初始設定 (Onboarding)]
* **介面元素：** 「每月總生活費」輸入框、「每月固定支出」輸入框、「夢想撲滿名稱與目標金額」輸入框、「開始生存」按鈕。
* **使用者行為：** 首次進入系統時填寫基本財務資料，進入首頁後開始每日生存。

### [功能區塊 B：生存首頁與動態交互 (主畫面)]
* **介面元素：** 巨大的「今日剩餘可用」金額、動態分佈進度條、一鍵記帳按鈕（喚起 Bottom Sheet）。
* **動態背景：** 餘額 > 30% 安全綠；餘額 < 30% 警戒橘；餘額 ≤ 0 破產紅。
* **使用者行為：** 點擊「記一筆」喚起面板，進行「手動輸入」。

### [功能區塊 C：自動換日結算與生存獎懲 (Automated Rollover)]
* **自動換日：** App 啟動時自動檢查 `last_login_date`。若偵測到缺失日期，系統自動執行逐日結算。
* **🏆 獎勵機制：** 達成 7 天連擊（未超支），獎勵今日額度 70% 金額存入夢想基金。
* **☠️ 懲罰機制：** 
  * 重超支沒收月預算 10% 罰金。
  * 違規費項（快樂水/社交）隔日加徵 20% 虛擬奢侈稅。

### [功能區塊 D：報表與大數據 (Insights)]
* **📊 生存報表 (Report)：** 本月總開銷統計與 CSS Conic-gradient 生存分類圓餅圖。
* **📋 詳細資料 (Details)：** 揭露精確的生存算式（總預算-固定支出-突發避險-已花費=可用）與系統即時參數。

## 4. 記帳的輸入與輸出規格 (Data I/O)

### 前端輸入有兩種型態：

* **型態一：手動輸入 (面板模式)**
  * **收支切換：** 支援「支出 (扣血)」與「收入 (補血)」。
  * **突發避險：** 標記為避險的交易不扣除今日額度，直接壓縮未來預算。

---

## 5. 頁面展示設計 (UI/UX Layout)

### 📱 底部導航系統 (5-Tab Navigation)
1. **📋 詳細 (Details)**：預算拆解公式與系統狀態。
2. **💰 撲滿 (Fund)**：圓環進度條、目標達成率與節餘歷史。
3. **🏠 主頁 (Home)**：核心大字、一鍵記帳、今日花銷預覽。
4. **📜 帳單 (History)**：分日期顯示清單，支援編輯/刪除。
5. **📊 報表 (Report)**：視覺化分类比例與開銷洞察。

---

## 6. 使用者操作流程 (User Flow)
* **Step 1：** 設定預算，系統產生每日生存基準。
* **Step 2：** 主頁面查看剩餘，點擊記一筆進行消費記錄。
* **Step 3：** 進行手動精確輸入，選擇分類並確認。
* **Step 4：** App 自動換日結算，存入節餘，維持生存連擊。
* **Step 5：** 前往報表頁面定期檢視漏財盲點。

---

## 7. 邊界條件與防呆機制 (Edge Cases)
* **極端超支：** 餘額歸零後鎖定背景為破產紅。
* **跨月重置：** 每月 1 號自動篩選新月份交易，重置預算分配。
