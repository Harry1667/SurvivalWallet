# [生存記帳 (Survival Wallet)] 產品需求文件 (PRD)

> **專案備註：** 本專案為個人自用 Web App (PWA)，注重生存遊戲化的理財體驗、極簡開發與行動端觸控優化。

---

## 1. 產品概述 (Project Overview)
* **核心價值：** 將記帳轉化為「生存遊戲」。不僅僅是記錄開銷，更是透過「今日可用 HP (預算)」來限制不必要的支出。每日結餘自動存入撲滿，超支則平攤至未來，讓存錢變成一場闖關遊戲。
* **目標對象：** 想存錢但缺乏自制力、對傳統瑣碎記帳感到疲憊的年輕族群。
* **使用情境：** 透過手機 PWA 快速開啟。消費後 3 秒內輸入，隨時查看「今天還能花多少」以決定是否進行下一筆非必要消費。

---

## 2. 技術架構 (Technical Architecture)
* **前端框架：** React.js (Vite) + TypeScript。
* **樣式系統：** Tailwind CSS v4 (採用 Modern CSS Variables 進行主題切換)。
* **動態特效：** Framer Motion (motion/react) 實作流暢的頁面轉場與 Modal 動畫。
* **資料持久化：** **Local-First SQLite**。使用 `sql.js` 在瀏覽器運行資料庫，並透過本地開發伺服器 API (`/api/db`) 實作檔案同步，確保資料不因重新整理而消失。
* **行動端優化：** 支援 PWA (Standalone 模式)、iOS Safe Area 適配、禁止縮放與長按反白，提供類原生 App 體驗。

---

## 3. 核心功能模組 (Core Features)

### [A. 初始化與設定 (Setup)]
* **Onboarding 流程：** 首次進入需填寫「每月總生活費」、「固定支出 (房租訂閱)」、「撲滿目標」。
* **動態計算：** 系統自動計算 `(總額 - 固定支出) / 本月剩餘天數` 作為每日 HP 基準。

### [B. 分頁系統 (5-Tab Navigation)]
1. **📋 詳細 (Details)**：
   * **生存儀表板**：顯示今日建議配額、月均日消耗、預估月底結餘。
   * **消費輪廓**：自動統計本月總筆數與最大支出分類。
   * **參數修改**：可直接在此更新預算與撲滿目標，即時重新計算。
2. **💰 撲滿 (Fund)**：
   * **夢想進度**：圓環進度條顯示目標達成率。
   * **歷史節餘**：查看每日自動結算存入的金額紀錄。
   * **手動提領**：支援將存下的錢拿出來緊急使用。
3. **🏠 主頁 (Home)**：
   * **核心 HP**：巨大的今日剩餘金額，具備壓力變色機制 (綠/橘/紅)。
   * **今日花銷**：卡片式列出今日所有交易，支援快捷編輯與刪除。
   * **一鍵記帳**：懸浮按鈕喚起 Bottom Sheet 快速輸入。
4. **📜 帳單 (History)**：
   * **全歷史清單**：按月份分日期排序，支援跨月查看與補記。
5. **📊 報表 (Report)**：
   * **視覺化分析**：動態圓餅圖展示開銷分布，協助發現漏財盲點。

### [C. 記帳邏輯 (Transaction Logic)]
* **收支切換**：支援「支出 (扣血)」與「收入 (補血)」。
* **突發避險 (Emergency)**：標記為避險的交易不扣除今日額度，而是直接重算本月剩餘預算。
* **奢侈稅機制**：針對特定分類（如：快樂水/零食）自動加徵 20% 虛擬奢侈稅，稅金自動存入撲滿。

### [D. 生存獎懲與結算 (Gamification)]
* **自動換日 (Rollover)**：啟動時自動比對日期。將昨日餘額歸零並轉存撲滿。
* **生存連擊 (Combo)**：連續未超支日數統計，未來可擴充勳章系統。
* **黑暗模式 (Dark Mode)**：內建手動切換開關，適配深夜記帳場景，採用黑晶玻璃風格。

---

## 4. UI/UX 設計規範
* **配色方案**：Emerald (安全)、Amber (警告)、Rose (危險)、Slate (沉穩底色)。
* **材質**：大量使用 Glassmorphism (毛玻璃) 結合背景模糊，創造剔透質感。
* **互動**：所有按鈕具備 `active:scale-95` 觸感回饋，Modal 具備 Spring 動畫。

---

## 5. 資料規格 (Data Schema)
* **user_settings**：存儲核心財務指標、連擊天數、懲罰分類 JSON、主題偏好。
* **transactions**：存儲 ID、金額、分類、標記、備註、建立時間、收支類型。
* **fund_records**：存儲基金的流入與流出紀錄。
