# [動態生存錢包 (Dynamic Survival Wallet)] Vibe Coding 專屬產品需求文件 (PRD)

> **專案備註：** 本專案為個人自用網站，注重極簡體驗、開發效率與遊戲化制約。

## 1. 產品概述 (Project Overview)
* **核心價值：** 專注於「今日可用額度」的極簡記帳，透過自動把每日餘額轉存撲滿、將超支平攤至未來，結合生存遊戲般的分類與獎懲機制，讓存錢變成無痛的全自動闖關遊戲。
* **目標對象：** 沒有記帳習慣、容易在月底透支、想存錢但缺乏自制力的大學生與年輕小資族。（本版本為開發者自用版）
* **使用情境：** 每天早上打開確認當天預算、消費後隨手花 3 秒鐘輸入金額、晚上睡前查看存錢進度與明日額度。

---

## 2. 系統架構與技術建議 (Tech Stack for AI)
* **前端呈現：** 建議使用 React.js (透過 Vite 建立) 或 React Native 搭配 Tailwind CSS。Tailwind 非常適合根據狀態（如：額度充足、快超支、破產警告）動態切換背景顏色與視覺 UI。
* **後端與資料庫：** 使用 **SQLite** 作為輕量級本地端資料庫儲存方案。無論是包裝成手機 App (如 React Native 搭配 SQLite 套件)，或是採用輕量後端 (Node.js/Python + SQLite)，都能提供穩定、結構化的資料查詢與持久化能力，適合處理關聯式與統計查詢。

## 3. 核心功能與邏輯 (Core Features - MVP)

### [功能區塊 A：初始設定 (Onboarding)]
* **介面元素：** 「每月總生活費」輸入框、「每月固定支出」輸入框、「夢想撲滿名稱與目標金額」輸入框、「開始生存」按鈕。
* **使用者行為：** 首次進入系統時填寫基本財務資料，填寫完成之後，點擊「開始生存」按鈕後進入首頁。

### [功能區塊 B：每日大聲公與記帳 (主畫面)]
* **介面元素：** 佔據畫面 **三分之一** 的巨大數字「今日剩餘可用：$XXX」、「記一筆」按鈕、依據狀態改變的全局背景色（綠/藍、橘、紅）。
* **使用者行為：** 點擊「記一筆」按鈕，會跳出一個視窗，讓使用者輸入消費金額與品項。輸入完成後，點擊「確認」按鈕即可完成記帳。若當日餘額 > 30% 顯示綠色；餘額 < 30% 顯示橘色。

### [功能區塊 C：每日結算機制與生存獎懲 (Gamification)]
* **介面元素：** 「夢想撲滿」進度條組件、一個隱藏版的「手動觸發換日」按鈕。
* **換日結算：** 系統偵測到跨越午夜 12 點時自動結算。若有剩餘，顯示撒花動畫並更新撲滿；若超支，顯示警告並扣減未來額度。
* **🏆 獎勵機制 (Rewards)：**
  * **連續生存解鎖：** 達成「連續 7 天未超支 (7-Day Streak)」，系統將這 7 天累積省下金額的 10% 獨立撥入「進階任務基金」（供自我投資或犒賞使用）。
  * **視覺稱號：** 根據連續達標天數，給予稱號徽章（如：🥉守財新手 ➡️ 🥈節流達人 ➡️ 🥇生存大師）。
☠️ 懲罰機制 (Penalties)：

未來額度重擊 (Budget Slash)： 若單日嚴重超支超過基準額度的 150%（且非避險支出），額外從「當月剩餘總預算」中沒收超支金額的 10% 作為罰金，導致未來每日額度急遽下降（吃土感）。

虛擬奢侈稅與夢想獻祭 (Luxury Tax & Dream Sacrifice)： 若昨日在特定非必要分類（如「🧋 快樂水/零食」）超支，今日該分類圖示會掛上紅色的 +20% Tax 標籤。

記帳表現： 當日若強行在該分類消費（例如花 60 元買飲料），系統資料庫 transactions 依然會如實記錄 60 元的花費，保持記帳數據乾淨。

扣款與資金流向： 在扣減「今日剩餘可用」時，系統會強制放大扣除 1.2 倍的金額（即扣除 72 元）。而那筆額外被徵收的 20% 罰金（12 元），系統會在背景自動將其加總至 user_settings 的 piggy_bank_saved（夢想撲滿）中。

心理機制： 透過直接壓縮當日的生存額度，讓你為衝動消費付出代價；但同時罰金轉化為夢想燃料，實現「用壞習慣強迫儲蓄」的終極目的。

### [功能區塊 D：破產預警與漏財雷達]
* **介面元素：** 紅色滿版警告彈窗（Red Alert）、首頁下方的浮動小提示框（Toast/Alert）。
* **破產預警：** 系統載入時計算 `(當月剩餘總額 / 過去7天平均日花費)`，若結果小於當月剩餘天數，觸發 UI 變紅與警告字眼。
* **漏財雷達：** 當特定非必要分類（如「快樂水/零食」）的當週加總金額 > 500 元時，觸發拿鐵因子提示字串。

### [功能區塊 E：突發避險機制 (Emergency Reserve)]
* **邏輯說明：** 針對醫療、車禍等不可抗力支出設立的防護網。
* **資料流向：** 標記為「避險」的消費**不計入**「今日剩餘可用」的扣減，也不會中斷連續未超支紀錄與觸發懲罰。系統會在背景直接從「當月剩餘總額」扣除，讓未來的每日基準額度平均縮水以維持財務平衡。

## 4. 記帳的輸入與輸出規格 (Data I/O & Database Schema)

### 前端輸入有兩種型態：

* **型態一：手動輸入 (彈窗模式)**
  * **金額 (Amount)：** 必填，呼叫純數字鍵盤 (`type="number"`，`inputMode="numeric"`)。禁止輸入負數。
  * **生存分類 (Category)：** 必填。包含以下 7 種遊戲化分類按鈕：
    1. 🍚 生存正餐 (維持生命所需)
    2. 🧋 快樂水/零食 (多巴胺來源，重點監控)
    3. 🧻 生活日用 (消耗品、理髮等)
    4. 🚌 交通通勤 (大眾運輸、加油)
    5. 🎉 娛樂社交 (聚餐、玩樂)
    6. 🧠 自我投資 (書籍、課程、軟硬體)
    7. 📦 其他雜項
  * **品項 (Item/Note)：** 選填，短文字輸入框 (`type="text"`)。
  * **突發避險標記 (Is Emergency)：** 選填，切換按鈕 (Toggle)。勾選時跳出二次確認。
  * **時間戳記 (Timestamp)：** 系統自動擷取。

* **型態二：AI 語音輸入**
  * **功能描述：** 語音講述今天花銷，接入 AI 語音辨識服務（Speech-to-Text + NLP），自動解析「金額」與「品項」，並**自動歸類**至上述 7 種生存分類中（例如聽到買咖啡自動歸入「快樂水/零食」），填入表單等待確認。

### 資料庫儲存結構 (SQLite Schema 建議)

**Table 1: `user_settings` (使用者設定與狀態)**
```sql
CREATE TABLE user_settings (
    id INTEGER PRIMARY KEY CHECK (id = 1),
    total_budget INTEGER NOT NULL,
    fixed_expenses INTEGER NOT NULL,
    daily_base_budget INTEGER NOT NULL,
    piggy_bank_name TEXT,
    piggy_bank_goal INTEGER,
    piggy_bank_saved INTEGER DEFAULT 0,
    current_daily_balance INTEGER,
    current_streak INTEGER DEFAULT 0,      -- 連續未超支天數
    last_login_date DATE
);
```

**Table 2: `transactions` (交易明細)**
```sql
CREATE TABLE transactions (
    id TEXT PRIMARY KEY,
    amount INTEGER NOT NULL,
    category TEXT NOT NULL CHECK(category IN ('生存正餐', '快樂水/零食', '生活日用', '交通通勤', '娛樂社交', '自我投資', '其他雜項')),
    is_emergency BOOLEAN DEFAULT 0,        -- 0: 正常消費, 1: 避險支出
    item TEXT DEFAULT '未分類消費',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 5. 頁面展示設計 (UI/UX Layout)
整體視覺主打「全螢幕沉浸感」與「狀態驅動 (State-driven)」，以 Mobile-First (手機端優先) 進行版面配置。

### 全局背景與動態主題色
* **安全期 (餘額 > 30%)：** 背景呈現柔和的薄荷綠或天空藍。
* **警戒期 (餘額 < 30%)：** 背景平滑過渡為警示的亮橘色。
* **破產期 (餘額 ≤ 0)：** 背景轉為強烈的血紅色，並伴隨畫面輕微震動的動畫特效。

### 由上至下的版面配置 (Top to Bottom)
1. **頂部資訊列 (Header bar)：**
   * 極簡無邊框設計。左上角顯示：「本月還剩 X 天」與當前稱號徽章。右上角顯示撲滿進度小圖示與金額。
2. **視覺核心區 (Hero Section)：**
   * 佔據螢幕上半部（約 33.3% 空間）。
   * 垂直水平置中顯示超大粗體數字：「$300」，上方為「今日剩餘可用」。
   * 大數字下方新增一條微型的多色進度條，顯示今日各「生存分類」的花費比例。
3. **記帳操作區 (Action Area)：**
   * 位於畫面中段。包含一個寬度 100%、帶有圓角的「記一筆」主要按鈕。
   * *(點擊後喚起彈窗，彈窗內包含金額、品項輸入、7 個分類 Icon 選擇與避險 Toggle 開關)*。
4. **底部動態面板 (Bottom Sheet / Recent Activity)：**
   * 畫面最下方顯示「今日最新紀錄」，撈取最新 3 筆顯示。
   * 觸發「漏財雷達」時，此區域上方浮現 Toast 提示框，3 秒後自動淡出。

## 6. 使用者操作流程 (User Flow)
* **Step 1：** 首次進入網站設定預算，系統算出每日基準額度為 300 元。
* **Step 2：** 進入首頁，綠色背景與巨大數字「$300」。吃完午餐，點擊「記一筆」輸入「$100」並選擇分類「🍚 生存正餐」，畫面數字即時滾動變成「$200」。
* **Step 3：** 下午想買飲料，用語音輸入「買了一杯珍奶 60 元」，AI 自動填入 $60 與「🧋 快樂水/零食」分類。若本週快樂水超標，跳出提示「💡 快樂水喝太多囉，東京機票提早買到！」。
* **Step 4：** 遇到突發感冒看醫生，輸入「$200」並勾選「⚠️ 突發避險」。今日剩餘不扣減，但未來每日可用額度會在背景重新計算並微幅下降。
* **Step 5：** 午夜 12 點跨日結算，當天餘額自動歸零並存入「東京撲滿」，進度條推進，隔天獲得全新的每日額度，連擊天數 (Streak) +1。

## 7. 邊界條件與防呆機制 (Edge Cases)
* **異常狀況 A (未填必填欄位)：** 在彈窗記帳時若金額為空、為負數，或未選擇分類，確認按鈕呈現 Disabled (禁用) 狀態。避險按鈕勾選時需二次確認。
* **異常狀況 B (單日極端超支)：** 若非避險花費過大，導致「未來每日基準額度」變成負數，首頁剩餘額度強制顯示「$0」，鎖定背景為紅色，提示「本月額度已徹底透支，請啟動緊急備用金」。
* **異常狀況 C (跨月換日處理)：** 每月 1 號自動跳出「新月份設定」彈窗，要求重新確認當月預算，重置天數與額度計算，保留撲滿累計金額與連擊天數。
